\begin{theory}{What is an algorithm?} \\
    Algorithms are a set of precisely defined steps are executed based off mechanical steps - there is no intelligence involved, just follow the steps given.
\end{theory}

\begin{aside}{Types of algorithms} \\
    \textit{1. Sequential deterministic algorithms:}
        \begin{itemize}
            \item Only one step can be executed at a time
            \item No element of randomness
        \end{itemize}
    \textit{2. Parallel algorithms:}
        \begin{itemize}
            \item Multiple steps can be ran at a time
            \item Probably not covered in 3121
        \end{itemize}
    \textit{3. Randomised algorithms:}
        \begin{itemize}
            \item Utilises random values
            \item Used in quantum computing
            \item Probably not covered in 3121
        \end{itemize}
\end{aside}

\begin{theory}{Why prove algorithms?} \\
    Sometimes, its not obvious if an algorithm does the things its advertised through common sense:
    \begin{itemize}
        \item Will your algorithm terminate in a timely manner?
        \item Can you prove that the algorithm won't infinitely loop?
        \item Is the time complexity as advertised?
    \end{itemize}
    Proofs are needed for these cases, most of the time they're the \textbf{only} way to know if the algorithm does its job.
\end{theory}

\begin{example}{Stable matching example} \\
    There are \textit{n} hospitals and \textit{n} doctors. Each hospital wants to hire exactly one doctor. Each doctor and hospital submit a list of preferences ranking which hospital they want to go to and which doctor they want.

    How do we design an algorithm where there is no hospital doctor pair (\textit{h, d}) that are dissatisfied with their result (stable):

    \begin{center}
        for pair $p = (h, d')$ \& $p' = (h', d)$\\
        \begin{itemize}
            \item $h$ prefers $d$ over $d'$
            \item $h'$ prefers $d'$ ovr $d$
        \end{itemize}
    \end{center}
    This implies that both hospitals and doctors have a preferred swap to make (unstable
    match) \par
    \bigskip
    \noindent\textbf{Stable matching algorithm} \par
    \noindent While there is a free hospital, pick a free hospital $h$ and offer a job to the highest
    ranking doctor $d$ on its preference list. \par
    \bigskip
    \noindent If $d$ doesn't have a job yet, $d$ always accepts the job.\par
    \par
    else if $d$ is already in a pair $(h' d)$:
    \begin{itemize}
        \item{if $h$ is higher ranked on the preference list for $d$, a new pair $(h, d)$ is
            created and the previous pair $(h', d)$ is deleted.}
        \item{if $h$ is ranked lower, then the offer is ignored.}\par
    \end{itemize}

    \noindent\textbf{How many ways to match hospital-doctors?} \par
    \noindent $n!$, first doctor has $n$ choices, next doctor has $n-1$ choices, so on and so forth.
    \par

    \noindent\textbf{Will a stable match always exist?} \par
    \noindent Yes, but it is not obvious - we need to use a proof:\par
    \par
    note that during the while loop:
    \begin{itemize}
        \item{for a doctor, the hospital they go to will always be higher on their ranks.}
        \item{a hospital will be paired with doctors of decreasing ranks.}
    \end{itemize}
    If we assume that the ranking is not stable: $p = (h, d)$ and $p' = (h', d')$ such that
    $h$ prefers $d'$ and $h'$ prefers $d$.\par
    \par
    Since $h$ prefers $d'$ to $d$, then it must have made an offer to $d'$ before offering to
    $d$, but since $h$ is paired with $d$, then $d'$ would need to have rejected $h$ because
    tehy were already at a hospital they prefer to $h$. This proves that its impossible to
    have $d'$ prefer $h'$ if $h$ is ranked higher.
\end{example}

\begin{theory}{Big O Notation} \\
    We say that $f(n) = O(g(n))$ if: \\
        \indent There are positive constants $C$ and $N$ such that \\
        \indent $0 \leq{f(n)} \leq{C\times{g(n)}}$ for all $n \geq{N}$.\\
    \\
    What does that acutally mean? lets take a look at some examples.\\
    \\
    Is $f(n) = O(g(n))$ for: \\
    \indent$f(n) = n + 5$, $g(n) = n^2-1$\\
    \\
    For this to hold true, there needs to be a value $n$ and $C$ such that
    every single value of $f(n) \leq{g(n)}$. \textbf{Is this true?}\\
    \\
    Well, given a sufficiently large $n$, yes! Lets prove it:\\
    Lets assign $C = 1$\\
    \begin{gather}
    f(n) = n + 5 < g(n) = n^2 - 1\\
    0 < n^2 - n - 6\\
    0 < (n - 3)(n + 2)\\
    \therefore n \text{ is true for } n > 3 \text{ or } n < -2
    \end{gather}
    \\this implies that for values $C = 1$ and every value $n > 3$, this equation holds true!\\
    In more human terms, we \textbf{need} to have a value $C$ that is multiplied with $g(n)$, 
    and there \textbf{needs} to exist a value $n$ such that every value greater than $n$ will satisfy
    \[
        0 \leq f(n) \leq C \times g(n).
    \]
    In doing so, we can say that the for $f(n)$ (which is the function of input size vs time) has
    a time complexity of $O(g(n))$ (keep in mind $g(n)$ will just be an equation that $f(n)$ will never
    exceed.)
\end{theory}

\begin{aside}{Is big-oh the be all-end all?}\\
    We know that big-oh demonstrates the worst case time complexities. But theres alot of
    data we're missing out on! What about best case? There are algorithms out there that
    can run faster than other algos even if they are bounded by the same big-oh time
    complexity! Think about running bubble sort on a sorted array with the first and last
    items swapped vs selection sort. This is where big theta and omega comes in.
\end{aside}

\begin{theory}{Big Omega}\\
    Big omega is the inverse of big-oh, it yields the asymtopic lower bound of a function.
    In plain words, $\Omega(f(n))$ is a metric that given a long enough running time,
    $f(n)$ will always be at least $\Omega(f(n))$. Another way to think about it is that
    given a time complexity $f(n)$, it will never be less than $\Omega(f(n))$ so plotting
    it on a graph will show that $\Omega(f(n))$ is bounded by $f(n)$.
\end{theory}

\begin{theory}{Big theta}\\
    given that we know $O(f(n))$ and $\Omega(f(n))$, we can combine those two graphs
    (where the space between big-oh and big omega represents big theta!). This means that
    the growth rate is the same in both best and worst case senarios.
\end{theory}

\begin{theory}{sum and product property}\\
    for all time complexities, we want to know how summing differnet time complexities and the product of them
    chagnes the total time complexity!

    for the sum of time complexities, we take the larger value: $$O(a) > O(b), O(a) + O(b)
    = O(a)$$. As an example, this makes sense! $$O(n) + O(n^2) = O(n^2)$$

    for the product of time complexities, the product of two time complexities will result
    in the product of the growth rates: $$O(a) \times O(b) = O(ab)$$.
\end{theory}

\begin{example}{Propositional Logic}\\
    \begin{itemize}
        \item Propositions are true/false statements. We can run operations on them!
        \item $\neg P$ is a negation of $P$, i.e. !P.
        \item $P \lor Q$ is $P$ OR $Q$, P $\|$ Q
        \item $P \land Q$ is $P$ AND $Q$, P $\&\&$ Q
        \item $P \Rightarrow Q$ is $P$ implies $Q$, i.e. if $P$ is true, then $Q$ is true. 
        \item $P \iff Q$ is $P$ if and only if $Q$ which means that for $P$ to true, $Q$
            also has to be true, and for $Q$ to be true, then $P$ also has to be true.
            People have explained iff as $P \Rightarrow Q, Q \Rightarrow P$, but I like to
            think of it as "they are either both true or they are either both false"
    \end{itemize}
\end{example}

\begin{theory}{Proof by induction}\\
    Proof by induction is pretty fun. Generally, induction is used to prove a sequence of
    things (that are ordered). If we prove that given any position, the next one holds a
    certain property and the first item also holds that property, then we can extrapolate
    that the entire set holds the property. For instance, for an infinitely long set of
    stairs that are green, if we are given the property that at any green step, the next
    step will be green and that the first step is green as well, then we can prove that
    the entire set of stairs are green!
\end{theory}
